## Goal
Create a new /powerflow section: SVG of 9-bus topology where each bus shows live voltage (pu) and each line shows live current (kA). On fault injection, animate transition from healthy steady-state → fault state using precomputed JSON; color-code lines by overcurrent severity.

## Acceptance Criteria
- [ ] New component `chronogrid-demo/src/components/PowerFlow.tsx` rendered in `App.tsx`
- [ ] Section loads `physics.json` once on mount, caches it, shows steady-state values when idle
- [ ] Each bus displays its voltage (e.g. `1.025 pu`) — color shifts amber when |V|<0.95 or >1.05
- [ ] Each line displays its current (e.g. `0.85 kA`) — color shifts red when >1.5× rated, animates flow direction
- [ ] Clicking a line opens fault-type picker (reuse existing pattern from Playground)
- [ ] On fault selection: smooth tween (≤500ms) of voltage/current values + color transitions, NO CNN inference call (this is pure physics view)
- [ ] "Reset to steady state" button restores idle view

## Notes
This is a separate page from `Playground.tsx` (which is the *ML* demo). PowerFlow is the *physics* demo. They should feel like complementary lenses on the same system.

UX expectation: idle state must visibly show numbers — users should understand "this is what normal operation looks like" before placing a fault.

## Implementation Plan
- [ ] Scaffold `PowerFlow.tsx` reusing the BUS/LINES topology constants from `Playground.tsx` (extract to shared `lib/topology.ts` if it makes sense)
- [ ] Add `usePhysics()` hook that fetches `/physics.json` and returns `{preFault, faults}`
- [ ] Bus voltage rendering: small label below each bus showing pu value, color via `voltageColor(v)`
- [ ] Line current rendering: midpoint label showing kA, animated dash speed proportional to current
- [ ] Use `framer-motion` `useMotionValue` for smooth value tweening between steady-state and fault state
- [ ] Color scale: green (normal) → amber (warning) → red (critical) using HSL interpolation
- [ ] Test: click each of the 6 lines × each fault type, confirm no NaN, no missing keys
